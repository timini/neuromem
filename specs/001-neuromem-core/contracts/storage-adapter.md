# Contract: `StorageAdapter` ABC

**Feature**: 001-neuromem-core
**Module**: `neuromem.storage.base`
**Scope**: The persistence contract. Any backend (SQLite, Postgres, Firebase, Qdrant, dict-in-memory mock) that implements `StorageAdapter` is a valid backing store for `NeuroMemory`.
**Binding**: This is the contract layer of Constitution Principle III (Layered, Modular, Pluggable Architecture). The orchestration layer imports this file and only this file from the storage package.

---

## ABC declaration

```python
from abc import ABC, abstractmethod
from typing import Any

class StorageAdapter(ABC):
    """
    The contract every persistence backend must satisfy for NeuroMemory to
    drive its cognitive loop. All 13 methods are @abstractmethod — no
    default implementations. Missing a method raises TypeError at
    instantiation time (see User Story 4 acceptance scenario #2).
    """
    ...
```

All methods below are decorated `@abstractmethod`.

---

## Method contracts

### Group 1 — Acquisition (Hippocampus)

#### `insert_memory(raw_content, summary, metadata=None) -> str`

```python
@abstractmethod
def insert_memory(
    self,
    raw_content: str,
    summary: str,
    metadata: dict | None = None,
) -> str: ...
```

**Preconditions**: `raw_content` is non-empty. `summary` may be an empty string (if a future design decision moves summary generation to the dreaming thread). `metadata`, if provided, is JSON-serialisable.

**Postconditions**:
- A new memory row has been persisted with `status='inbox'`, `access_weight=1.0`, `created_at=int(time.time())`, `last_accessed=NULL`.
- Returns the new memory's UUID string (generated by the adapter via `uuid.uuid4()`).
- Is atomic: either the memory exists fully after return or not at all.

**Errors**: `StorageError` on I/O failure. `ValueError` on `metadata` that won't JSON-serialise.

---

#### `count_memories_by_status(status) -> int`

```python
@abstractmethod
def count_memories_by_status(self, status: str) -> int: ...
```

**Preconditions**: `status` is one of `'inbox' | 'dreaming' | 'consolidated' | 'archived'`.

**Postconditions**: Returns the count of rows in `memories` with the given `status`. Must be O(log n) or better (use the `idx_memories_status` index). Called on every `enqueue()` to check the threshold, so must be cheap.

**Errors**: `ValueError` on unknown status.

---

#### `get_memories_by_status(status, limit=None) -> list[dict]`

```python
@abstractmethod
def get_memories_by_status(
    self,
    status: str,
    limit: int | None = None,
) -> list[dict[str, Any]]: ...
```

**Preconditions**: `status` is valid. `limit`, if provided, is a positive integer.

**Postconditions**: Returns up to `limit` memory records (or all if `limit is None`) matching the given status. Each dict has the full shape described in `data-model.md §Memory`. Order is unspecified in v1 but should be deterministic within a single call for test stability (recommended: `ORDER BY created_at ASC`).

**Errors**: `ValueError` on invalid status or non-positive limit.

---

#### `update_memory_status(memory_ids, new_status) -> None`

```python
@abstractmethod
def update_memory_status(
    self,
    memory_ids: list[str],
    new_status: str,
) -> None: ...
```

**Preconditions**: `memory_ids` is a list of valid UUID strings (may be empty, which is a no-op). `new_status` is one of the four legal statuses AND represents a legal transition from every target's current status (see data-model.md §State Machine). The adapter does **not** validate legal transitions in v1 — that's orchestration's responsibility.

**Postconditions**: Every matching row now has `status = new_status`. The update is atomic (single transaction for all IDs). This is the method used by the double-buffer pattern — it MUST be callable in a single SQL statement to preserve atomicity.

**Errors**: `StorageError` on I/O failure. `ValueError` on invalid status.

---

#### `get_memory_by_id(memory_id) -> dict | None`

```python
@abstractmethod
def get_memory_by_id(self, memory_id: str) -> dict[str, Any] | None: ...
```

**Preconditions**: `memory_id` is a string (need not be a valid UUID — unknown IDs return `None`, not an error).

**Postconditions**: Returns a full memory dict if found, `None` otherwise. Missing is not an error — `retrieve_memories()` relies on silent skip (User Story 2 acceptance scenario #3).

### Group 2 — Consolidation (Neocortex / Graph)

#### `upsert_node(node_id, label, embedding, is_centroid) -> None`

```python
import numpy as np
from numpy.typing import NDArray

@abstractmethod
def upsert_node(
    self,
    node_id: str,
    label: str,
    embedding: NDArray[np.floating] | list[float],
    is_centroid: bool,
) -> None: ...
```

**Preconditions**:
- `node_id` is a UUID string (may be pre-generated by the caller so centroid IDs can be stable).
- `label` is a non-empty string. For centroids, single-word.
- `embedding` is a non-empty numpy array or Python list of floats with the same length as every other node currently in storage. Python lists are converted to `np.ndarray` (float32) by the adapter before serialisation.
- `is_centroid` is a `bool`.

**Postconditions**:
- If a row with `id = node_id` exists, its `label`, `embedding`, and `is_centroid` are updated.
- Otherwise a new row is inserted.
- Atomic.

**Errors**:
- `ValueError` if `embedding` is empty or its length differs from existing nodes in storage (v1 rule — see data-model.md V-N1).
- `StorageError` on I/O failure.

**Performance warning**: Per data-model.md V-N3, the `SQLiteAdapter` logs a `WARNING` at the `neuromem` logger if the incoming embedding has more than 4096 dimensions.

---

#### `get_all_nodes() -> list[dict]`

```python
@abstractmethod
def get_all_nodes(self) -> list[dict[str, Any]]: ...
```

**Postconditions**:
- Returns every node in storage as a list of dicts `{"id", "label", "embedding", "is_centroid"}`.
- `embedding` is deserialised back to `list[float]` — the caller receives vectors, not serialised blobs.
- Order is unspecified but deterministic within a single call (recommended: `ORDER BY id`).

**Performance note**: This is the hot path for the dreaming cycle (called once per cycle to build the full tag list for clustering) and for `SQLiteAdapter.get_nearest_nodes()` (called on every `search_memory()` call). Native-vector adapters should override `get_nearest_nodes()` to avoid this full-table scan.

---

#### `insert_edge(source_id, target_id, weight, relationship) -> None`

```python
@abstractmethod
def insert_edge(
    self,
    source_id: str,
    target_id: str,
    weight: float,
    relationship: str,
) -> None: ...
```

**Preconditions**:
- `source_id` and `target_id` are strings. They need not reference existing rows (v1 has no foreign-key validation — see data-model.md V-E3).
- `weight` is a float. Conventionally in `[0.0, 1.0]`.
- `relationship` is one of `'has_tag' | 'child_of'`.

**Postconditions**:
- If an edge with the same `(source_id, target_id, relationship)` triple exists, the call is a silent no-op (idempotent insert — the SQLite adapter uses `INSERT OR IGNORE`).
- Otherwise a new row is inserted.

**Errors**: `ValueError` on unknown relationship. `StorageError` on I/O failure.

---

#### `remove_edges_for_memory(memory_id) -> None`

```python
@abstractmethod
def remove_edges_for_memory(self, memory_id: str) -> None: ...
```

**Preconditions**: `memory_id` is a UUID string. Non-existent IDs are silent no-ops.

**Postconditions**: All `has_tag` edges with `source_id = memory_id` have been removed from the edges table. Called by the archival step in Phase C.

**Note**: Does NOT remove any `child_of` edges (those live between nodes, not between memories). Does NOT delete the tag nodes themselves — they may still be referenced by other non-archived memories.

### Group 3 — Recall (Prefrontal Cortex)

#### `get_nearest_nodes(query_embedding, top_k=5) -> list[dict]`

```python
import numpy as np
from numpy.typing import NDArray

@abstractmethod
def get_nearest_nodes(
    self,
    query_embedding: NDArray[np.floating] | list[float],
    top_k: int = 5,
) -> list[dict[str, Any]]: ...
```

**Preconditions**:
- `query_embedding` is a non-empty numpy array or Python list of floats with length matching every stored node's embedding.
- `top_k >= 1`.

**Postconditions**:
- Returns the `top_k` nodes (or fewer if the graph has fewer than `top_k` nodes) sorted by decreasing cosine similarity to `query_embedding`.
- Each returned dict has `{"id", "label", "embedding", "is_centroid", "similarity"}` where `embedding` is a `np.ndarray` (not a raw blob or list) and `similarity` is the computed cosine value (added by this method; it's not stored on the row itself).
- Nodes connected only to archived memories are still returned — filtering by memory reachability happens in `get_subgraph`, not here.

**Implementation strategies**:
- **SQLiteAdapter (v1 default)**: `SELECT id, label, embedding, embedding_dim, is_centroid FROM nodes` → stack all embeddings into one `(N, D)` `np.float32` matrix via `np.frombuffer(...).reshape(1, -1)` then `np.vstack` (or `np.stack`) → call `batch_cosine_similarity(query, matrix)` from `neuromem.vectors` → `np.argpartition` for top-k → sort the top-k by similarity descending → return. The matrix build is ~60 MB in RAM for 10K × 1536-dim and ~15–25 ms on a modest laptop. SC-003 budget is 500 ms, so we have comfortable headroom for adjacent work (filter, sort, render).
- **Native-vector adapter (future)**: Delegate to DB-level ANN search. `SELECT id, label, embedding, 1 - (embedding <-> :q) AS sim FROM nodes ORDER BY sim DESC LIMIT :k` (pgvector syntax).

**Errors**: `ValueError` on empty `query_embedding` or dimension mismatch. `StorageError` on I/O failure.

#### `get_subgraph(root_node_ids, depth=2) -> dict`

```python
@abstractmethod
def get_subgraph(
    self,
    root_node_ids: list[str],
    depth: int = 2,
) -> dict[str, Any]: ...
```

**Preconditions**: `root_node_ids` is a non-empty list of node UUID strings. `depth >= 0`.

**Postconditions**:
- Returns a dict:
  ```python
  {
      "nodes": list[dict],     # every node within `depth` hops of a root,
                               # deduplicated, full records
      "edges": list[dict],     # every edge whose source or target is in `nodes`
      "memories": list[dict],  # every non-archived memory reachable by a
                               # has_tag edge from any node in `nodes`
  }
  ```
- Memories with `status = 'archived'` are EXCLUDED from the `memories` list (archival strips them from the active graph, spec §Resolved Design Decisions implication).
- A node may appear in `nodes` exactly once regardless of how many parents reach it — the tree-renderer in `ContextHelper` handles multi-parent nodes by noting `(also under: …)`.

**Traversal**: Starts from `root_node_ids`, follows `child_of` edges in **both directions** (centroid → child and child → centroid-parent), up to `depth` hops. Then collects `has_tag` edges from every reached node to gather memories.

**Errors**: `StorageError` on I/O failure.

### Group 4 — Forgetting (Synaptic Pruning)

#### `apply_decay_and_archive(decay_lambda, archive_threshold, current_timestamp) -> list[str]`

```python
@abstractmethod
def apply_decay_and_archive(
    self,
    decay_lambda: float,
    archive_threshold: float,
    current_timestamp: int,
) -> list[str]: ...
```

**Preconditions**:
- `decay_lambda > 0`.
- `0 <= archive_threshold < 1.0`.
- `current_timestamp` is a Unix epoch seconds integer.

**Postconditions**:
- For every memory with `status = 'consolidated'`:
  - Compute `t = current_timestamp - last_accessed` (use `created_at` if `last_accessed` is NULL).
  - Compute `W_new = access_weight * exp(-decay_lambda * t)`.
  - Update `access_weight = W_new`.
  - If `W_new < archive_threshold`:
    - Set `status = 'archived'`.
    - Call `remove_edges_for_memory(memory.id)`.
- Returns the list of memory IDs that were archived in this call.
- Atomic per-memory (each memory's decay + archival is one transaction; the whole method may use a single transaction if the adapter supports it).

**Errors**: `StorageError` on I/O failure. `ValueError` on invalid parameters.

---

#### `spike_access_weight(memory_ids, timestamp) -> None`

```python
@abstractmethod
def spike_access_weight(
    self,
    memory_ids: list[str],
    timestamp: int,
) -> None: ...
```

**Preconditions**: `memory_ids` is a list of UUID strings (empty list is a no-op). `timestamp` is a Unix epoch seconds integer.

**Postconditions**:
- For every memory in `memory_ids` with `status = 'consolidated'`:
  - `access_weight = 1.0`
  - `last_accessed = timestamp`
- Memories with other statuses are silently skipped (no LTP for inbox/dreaming/archived — see data-model.md V-M4, V-M5).

**Errors**: `StorageError` on I/O failure.

---

## Error types the adapter is allowed to raise

- **`StorageError`** — a `RuntimeError` subclass defined in `neuromem.storage.base` for any I/O, disk, or database-level failure. Callers should treat this as "the adapter is broken, stop". Not recoverable in v1.
- **`ValueError`** — for caller-supplied argument problems (bad status, empty vector, etc.).
- **`TypeError`** — only from ABC instantiation. Concrete adapters should not raise `TypeError` from methods.

## Contract tests

Every adapter implementation MUST pass the following contract tests (to land in `tests/test_storage_sqlite.py` and be parameterised for future adapters):

1. **Construction**: Instantiate with a reasonable config, confirm schema is initialised (implicit: calls succeed without NO_SUCH_TABLE).
2. **insert_memory round-trip**: `insert_memory("text", "summary", {"k": "v"})` → `get_memory_by_id(returned_id)` returns a dict with the fields populated and `status='inbox'`, `access_weight=1.0`.
3. **Status state machine**: `insert_memory` → `update_memory_status([id], 'dreaming')` → `update_memory_status([id], 'consolidated')` → every step reflected in `get_memory_by_id`.
4. **count_memories_by_status**: Insert 3 memories, assert `count_memories_by_status('inbox') == 3`. After flipping 2 to `'dreaming'`, assert `2` and `1` respectively.
5. **Node upsert**: `upsert_node(id1, "Python", [0.1]*16, False)` → `get_all_nodes()` contains that node with the exact embedding.
6. **Node dimension mismatch**: Insert a node with `[0.1]*16`, then try to insert another with `[0.1]*32`. Expect `ValueError`.
7. **Edge insert idempotency**: `insert_edge(a, b, 0.9, 'has_tag')` twice → only one row exists.
8. **get_nearest_nodes**: Insert 5 nodes with known embeddings, query with a vector close to one of them, assert that node ranks first.
9. **get_subgraph**: Build a small tree (centroid → leaf1, centroid → leaf2, leaf1 ← memoryA), query with root = centroid, assert `memoryA` appears in `memories`.
10. **apply_decay_and_archive**: Insert a consolidated memory with `last_accessed` set 60 days ago, run decay with `λ=3e-7`, assert the memory is archived and its edges are removed.
11. **spike_access_weight ignores non-consolidated**: Insert an `inbox` memory, call `spike_access_weight([id], now)`, assert `access_weight` is still `1.0` (unchanged from the initial default) and `last_accessed` is still `NULL`.
12. **Archival preserves content**: Insert → archive → `get_memory_by_id` still returns the original `raw_content` and `summary`.
13. **Missing ID silent skip**: `get_memory_by_id("nonexistent-id") → None`. `spike_access_weight(["nonexistent-id"], now)` → no error.

These tests are written **first** (red-green-refactor, Principle V). The adapter implementation follows.
